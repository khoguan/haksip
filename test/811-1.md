### 祕訣：gh API mode md to html converter

Interactive mode: `gh markdown-preview`

API mode to generate html from md:

```shell
gh api markdown --field text="$(cat chinese-text-test.md)" --field mode=gfm > rendered.html
```

### 干焦 HTML comment 會使當做 continuation marker

```markdown
一二三<!--
-->四五六<!--
-->七八九
```

gh markdown-preview 就會變做：

> 一二三四五六七八九

### Warp AI reply

Based on my research, markdownlint-cli2 does NOT have a built-in rule to enforce
single spaces between regular words in text content. Here's what I found:

What markdownlint-cli2 DOES handle:

1. MD019 - Fixes multiple spaces after # in headings (e.g., ## heading → ##
   heading)
2. MD021 - Fixes multiple spaces in closed ATX headings (e.g., # heading #)
3. MD027 - Fixes multiple spaces after blockquote symbols (e.g., > text → >
   text)
4. MD030 - Handles spaces after list markers
5. MD009 - Removes trailing spaces at end of lines

What it DOESN'T handle:

As you correctly observed, markdownlint-cli2 will fix ## this way to ## this
way - it only fixes the spaces after the ## marker but leaves the multiple
spaces between words intact. There is no built-in rule to enforce single spaces
between words in regular text content.

Workarounds:

If you need to enforce single spaces between words, you have a few options:

1. Custom rule: Write a custom markdownlint rule to detect and fix multiple
   spaces between words
2. Pre/post-processing: Use a separate tool like sed or awk to fix multiple
   spaces
3. Prettier: Use Prettier with markdown support, which will normalize spaces
   between words:

Custom rule: Write a custom markdownlint rule to detect and fix multiple spaces
between words Pre/post-processing: Use a separate tool like sed or awk to fix
multiple spaces Prettier: Use Prettier with markdown support, which will
normalize spaces between words:

The reason markdownlint doesn't have this rule is likely because in HTML
rendering, multiple spaces are collapsed to single spaces anyway, so it's not
