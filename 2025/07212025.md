Quick Setup Example (dict.nvim):

You can add this to your plugins configuration:

```lua
{
  "kkharji/dict.nvim",
  config = function()
    require("dict").setup({
      -- Configuration options
      dict_server = "dict://dict.org:2628", -- Default dictionary server
      keymaps = {
        ["<leader>wd"] = {
          mode = { "n", "v" },
          desc = "Look up word definition"
        }
      }
    })
  end
}
```

Most of these plugins allow you to:
-  Look up the word under cursor with a keymap (usually <leader>wd or similar)
-  Search for definitions in visual mode
-  Use offline dictionaries or online APIs
-  Display results in floating windows or split buffers

### Warp AI `dictinary.lua` 建置報告

Dictionary Plugin Features

The dictionary plugin provides the following functionality:

Keymaps:
-  <leader>wd - Look up word under cursor (works in both normal and visual mode)
-  <leader>wD - Look up word with input prompt
-  <leader>ws - Look up synonyms for word under cursor

How it works:
1. Uses your installed dict command via Homebrew
2. Opens definitions in a split window
3. Results are displayed in a read-only buffer
4. Press q to close the dictionary window

What's included:
-  Word lookup - Get definitions for the word under cursor
-  Custom word lookup - Enter any word to look up
-  Visual selection - Select text and look it up
-  Synonyms - Try to find synonyms using thesaurus databases
-  Error handling - Proper notifications for missing words or connection issues

Usage:
1. Basic usage: Place cursor on any word and press <leader>wd
2. Visual mode: Select text and press <leader>wd  
3. Custom lookup: Press <leader>wD and type any word
4. Synonyms: Press <leader>ws on any word

The plugin integrates with your existing which-key setup, so when you press
<leader>w, you'll see all the dictionary options with their descriptions.

To use the dictionary plugin, restart Neovim or run :source % while in the
dictionary.lua file. Then try placing your cursor on any word and pressing
<leader>wd!
